with_accumulation<BaseGenerator,T1,T2,AccumulationKey>

Description

In addition to the other operations it inherits from the return type of BaseGenerator, this Tree Node adaptor performs statistical computations on user data when the node's structure and/or contents are modified, then stores the result.

Important

Optional Boost.Accumulators parameters are currently unsupported. This may change once this data structure can detect the availability of Boost.Parameter keywords used by Boost.Accumulators features.

Important

Weighted samples are currently unsupported.

Example

A GUI application that renders row numbers needs to know, in sublinear time, which variable-height row numbers should be rendered on-screen. Independently, the application must also compute the minimum priority of each row number in terms of its own priority and that of its sub-rows.

GUITable associates the example key types with the user data that our program will need. HeightSumKey is a template instantiation of tree_node::accumulation_key that will instruct the corresponding tree_node::with_accumulation_base template instantiation to store the sum of the example_keys::row_height values of a node and its children, while MinPriorityKey is a template instantiation of tree_node::accumulation_key that will instruct the corresponding with_accumulation template instantiation to store the minimum of the example_keys::row_priority values of a node and its children.

GUIBinaryCompare is a Binary Predicate model that implements two function call operators. The first one determines whether or not tree_node::binary_descendant() should traverse the left branch; the second one determines whether or not the algorithm should traverse the right branch. 

namespace example_keys {

    struct row_number
    {
    };

    struct row_height
    {
    };

    struct row_priority
    {
    };
}

typedef boost::fusion::map<
            boost::fusion::pair<example_keys::row_number,unsigned int>
          , boost::fusion::pair<example_keys::row_height,unsigned int>
          , boost::fusion::pair<example_keys::row_priority,unsigned int>
        >
        GUITable;
typedef boost::tree_node::accumulation_key<example_keys::row_height>
        HeightSumKey;
typedef boost::tree_node::accumulation_key<
            example_keys::row_priority
          , boost::accumulators::tag::min
        >
        MinPriorityKey;

class GUIBinaryCompare
{
    mutable unsigned int total_height;

 public:
    GUIBinaryCompare() : total_height(0)
    {
    }

    template <typename Node>
    bool operator()(unsigned int value, Node const& node) const
    {
        typename Node::const_pointer left = node.get_left_child_ptr();

        return left && (
            value < this->total_height + get(*left, HeightSumKey())
        );
    }

    template <typename Node>
    bool operator()(Node const& node, unsigned int value) const
    {
        this->total_height += get(node, example_keys::row_height());

        typename Node::const_pointer left = node.get_left_child_ptr();

        if (left)
        {
            this->total_height += get(*left, HeightSumKey());
        }

        return this->total_height < value;
    }
};

With the boilerplate code out of the way, we can define the GUINode data structure. The tree_node::binary_node_base_gen type induces GUINode to model the Binary Tree Node concept, while the tree_node::with_accumulation_base and with_accumulation template instantiations augment GUINode with the values associated with HeightSumKey and MinPriorityKey, respectively. 

typedef boost::tree_node::with_accumulation<
            boost::tree_node::with_accumulation_base_gen<
                boost::tree_node::binary_node_base_gen<>
              , HeightSumKey
            >
          , GUITable
          , void
          , MinPriorityKey
        >
        GUINode;

Using the tree_node::binary_descendant() algorithm with GUIBinaryCompare enables the application to find the row number that occupies the space a given number of pixels from the top in logarithmic time. A traversal through the nodes will also reveal that the tree exhibits the heap-order property with respect to MinPriorityKey values.
