The BOOST_PP_ASSERT_IS_LIST macro conditionally causes a preprocessing error if its parameter is not a list.
Although the macro itself is not a variadic macro, it needs variadic macro support in order to be invoked.

Usage

BOOST_PP_ASSERT_IS_LIST(param) (v)

Arguments

param
  A parameter to check for a list..

Remarks

If param is not a list, this macro causes a preprocessing error.  Otherwise, it expands to nothing..

See Also

  • BOOST_PP_ASSERT_IS_ARRAY
  • BOOST_PP_ASSERT_IS_SEQ
  • BOOST_PP_ASSERT_IS_TUPLE

Requirements

Header:  <boost/preprocessor/debug/assert_is_list.hpp>

Sample Code

#include <boost/preprocessor/debug/assert_is_list.hpp>

#define A_LIST (a,(b,(c,BOOST_PP_NIL)))
#define NOT_JUST_LIST (a,(b,(c,BOOST_PP_NIL))) text
#define AN_ARRAY (3,(a,b,c))
#define A_TUPLE (a,b,c)
#define A_SEQ (a)(b)(c)
#define A_TEXT text

BOOST_PP_ASSERT_IS_LIST(A_LIST) // no output
BOOST_PP_ASSERT_IS_LIST(NOT_JUST_LIST) // generates preprocessing error
BOOST_PP_ASSERT_IS_LIST(AN_ARRAY) // generates preprocessing error
BOOST_PP_ASSERT_IS_LIST(A_TUPLE) // generates preprocessing error
BOOST_PP_ASSERT_IS_LIST(A_SEQ) // generates preprocessing error
BOOST_PP_ASSERT_IS_LIST(A_TEXT) // generates preprocessing error
