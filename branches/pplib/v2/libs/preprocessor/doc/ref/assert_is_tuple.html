The BOOST_PP_ASSERT_IS_TUPLE macro conditionally causes a preprocessing error if its parameter is not a tuple.
Although the macro itself is not a variadic macro, it needs variadic macro support in order to be invoked.

Usage

BOOST_PP_ASSERT_IS_TUPLE(param) (v)

Arguments

param
A parameter to check for a tuple.

Remarks

If param is not a tuple, this macro causes a preprocessing error.  Otherwise, it expands to nothing.
An array, a list, and a sequence of size 1 are also tuples.

See Also

  • BOOST_PP_ASSERT_IS_ARRAY
  • BOOST_PP_ASSERT_IS_LIST
  • BOOST_PP_ASSERT_IS_SEQ

Requirements

Header:  <boost/preprocessor/debug/assert_is_tuple.hpp>

Sample Code

#include <boost/preprocessor/debug/assert_is_tuple.hpp>

#define A_TUPLE (a,b,c)
#define NOT_JUST_TUPLE (a,b,c) text
#define AN_ARRAY (3,(a,b,c))
#define A_LIST (a,(b,(c,BOOST_PP_NIL)))
#define A_SEQ (a)(b)(c)
define A_SEQ_OF_SIZE_ONE (a)
#define A_TEXT text

BOOST_PP_ASSERT_IS_TUPLE(A_TUPLE) // no output
BOOST_PP_ASSERT_IS_TUPLE(NOT_JUST_TUPLE) // generates preprocessing error
BOOST_PP_ASSERT_IS_TUPLE(AN_ARRAY) // no output, an array is also a tuple
BOOST_PP_ASSERT_IS_TUPLE(A_LIST) // no output, a list is also a tuple
BOOST_PP_ASSERT_IS_TUPLE(A_SEQ) // generates preprocessing error
BOOST_PP_ASSERT_IS_TUPLE(A_SEQ_OF_SIZE_ONE) // no output, a sequence of size 1 is also a tuple
BOOST_PP_ASSERT_IS_TUPLE(A_TEXT) // generates preprocessing error
