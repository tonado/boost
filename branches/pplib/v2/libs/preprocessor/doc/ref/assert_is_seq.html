The BOOST_PP_ASSERT_IS_SEQ macro conditionally causes a preprocessing error if its parameter is not a sequence.
Although the macro itself is not a variadic macro, it needs variadic macro support in order to be invoked.

Usage

BOOST_PP_ASSERT_IS_SEQ(param) (v)

Arguments

param
A parameter to check for a sequence.

Remarks

If param is not a sequence, this macro causes a preprocessing error.  Otherwise, it expands to nothing.
A sequence consists of one or more adjacent parenthesized elements. Therefore a tuple whose size is 1 is also a sequence.

See Also

  • BOOST_PP_ASSERT_IS_ARRAY
  • BOOST_PP_ASSERT_IS_LIST
  • BOOST_PP_ASSERT_IS_TUPLE

Requirements

Header:  <boost/preprocessor/debug/assert_is_seq.hpp>

Sample Code

#include <boost/preprocessor/debug/assert_is_seq.hpp>

#define A_SEQ (a)(b)(c)
#define A_ONE_ELEMENT_TUPLE (a)
#define NOT_JUST_SEQ (a)(b)(c) text
#define AN_ARRAY (3,(a,b,c))
#define A_TUPLE (a,b,c)
#define A_LIST (a,(b,(c,BOOST_PP_NIL)))
#define A_TEXT text

BOOST_PP_ASSERT_IS_SEQ(A_SEQ) // no output
BOOST_PP_ASSERT_IS_SEQ(A_ONE_ELEMENT_TUPLE) // no output, a one element tuple is also a sequence
BOOST_PP_ASSERT_IS_SEQ(NOT_JUST_SEQ) // generates preprocessing error
BOOST_PP_ASSERT_IS_SEQ(AN_ARRAY) // generates preprocessing error
BOOST_PP_ASSERT_IS_SEQ(A_TUPLE) // generates preprocessing error
BOOST_PP_ASSERT_IS_SEQ(A_LIST) // generates preprocessing error
BOOST_PP_ASSERT_IS_SEQ(A_TEXT) // generates preprocessing error
