The BOOST_PP_IS_TUPLE_BEGIN macro tests whether its parameter begins with a tuple. Although the macro itself is not a variadic macro, it needs variadic macro support in order to be invoked.

Usage

BOOST_PP_IS_TUPLE_BEGIN(param) (v)

Arguments

param
The parameter to test.

Remarks

If the param begins with a tuple the macro returns 1, otherwise returns 0. The param may contain other tokens after the tuple and it still will return 1. Therefore this macro is NOT a test whether or not the param is a tuple or not, since there is absolutely no safe way to test that. In order to assert that the param is a tuple, use BOOST_PP_ASSERT_IS_TUPLE.

Requirements
Header:  <boost/preprocessor/facilities/is_tuple_begin.hpp>

Sample Code

#include <boost/preprocessor/facilities/is_tuple_begin.hpp>
#include <boost/preprocessor/debug/assert_is_tuple.hpp>

#define ONLY_TUPLE (a,b,c)
#define STARTS_WITH_TUPLE (a,b,c) cheers
#define ENDS_WITH_TUPLE cheers (a,b,c)
#define HAS_NO_TUPLE cheers

BOOST_PP_IS_TUPLE_BEGIN(ONLY_TUPLE) // output is 1
BOOST_PP_IS_TUPLE_BEGIN(STARTS_WITH_TUPLE) // output is 1
BOOST_PP_IS_TUPLE_BEGIN(ENDS_WITH_TUPLE) // output is 0
BOOST_PP_IS_TUPLEv(HAS_NO_TUPLE) // output is 0

BOOST_PP_ASSERT_IS_TUPLE(ONLY_TUPLE) // no output
BOOST_PP_ASSERT_IS_TUPLE(STARTS_WITH_TUPLE) // generates a preprocessing error
BOOST_PP_ASSERT_IS_TUPLE(ENDS_WITH_TUPLE) // generates a preprocessing error
BOOST_PP_ASSERT_IS_TUPLE(HAS_NO_TUPLE) // generates a preprocessing error
